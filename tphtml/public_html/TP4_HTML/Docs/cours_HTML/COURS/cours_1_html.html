HTML 5 – Partie 1

Historique des versions

Le HTML 5 (octobre 2014) puis HTML 5.1 (octobre 2017) succède à HTML 4.1 (décembre 1999), qui lui-même succède à la version 4 (avril 1998).

L’objectif du cours n’est pas de faire un comparatif entre les différentes versions.

Le W3C (World Wide Web Consortium) est un organisme international qui développe des standards pour le Web. Actuellement, de nombreux standards sont élaborés par cet organisme : XML, CSS, SVG… et HTML.

Le W3C travaille sur le mandat de ses membres parmi lesquelles se trouvent les grands noms de l’informatique : Google, Mozilla, Microsoft, HP, Cisco ….

En clair : aucune technologie n’obtient le cachet de W3C sans être approuvée par la communauté du Web.

Les membres délèguent des ingénieurs au sein de W3C et participent ainsi à l’élaboration des spécifications techniques pour les technologies du Web.

Le HTML 5 et tous les autres standards sont issus de ce travail.

Ainsi une page WEB écrite en HTML 5, qui respecte les recommandations du W3C sera interprétée de la même façon, quel que soit le navigateur (Chrome, IE, Edge, Firefox, Safari, Opera…)

Le W3C propose le service Markup Validation Service à l’adresse : https://validator.w3.org. Ce service permet de vérifier que le code HTML d’une page respecte le standard.

Le W3C recommande HTML5 depuis 2014.

Qu’est-ce que le HTML ?

HTML signifie Hyper Text Markup Language : Langage de balisage d’hypertexte

Permet de définir des documents Web, reliés entre eux par des liens, contenant :

  • Du texte

  • Des images

  • Du son, des vidéos

C’est un langage de description de documents. Il n’a pas pour objectif la mise en forme des documents. Ce qui permet la mise en forme d’un document c’est le CSS que nous étudierons plus tard dans ce cours.

Conclusion : on utilise HTML pour décrire le fond, et CSS pour la forme.

La notion de web sémantique

Les pages Web que nous rédigeons sont destinées à être lues par des humains, non par des machines. Néanmoins des ordinateurs spécialisés sont chargés d’analyser les pages Web dans l’objectif de les référencer. C’est le cas des moteurs de recherche comme Google. Pour analyser vos pages, ces derniers doivent comprendre le sens du contenu.

Définition Wikipédia : le Web sémantique est un système qui permet aux machines de « comprendre » et de répondre aux demandes complexes de l’homme en fonction du sens de ces demandes. Une telle « compréhension » exige que les sources d’information pertinentes aient été sémantiquement structurées au préalable.

Autre définition Wikipédia : L’optimisation pour les moteurs de recherche, où SEO (pour search engine optimization) est un ensemble de techniques pour optimiser la visibilité d’une page Web dans les pages de résultats de recherche. Ces techniques cherchent à améliorer la compréhension par les robots d’indexation de la thématique et du contenu d’une ou de l’ensemble des pages d’un site Web et à augmenter le trafic naturel du site.

L’objectif est d’améliorer le positionnement d’une page Web dans les pages de résultats de recherche sur des mots-clés ciblés (selon les thèmes principaux du site). On considère que le positionnement d’un site est bon lorsqu’il est classé dans la première page des résultats de recherche, dans l’une des dix premières réponses naturelles d’une recherche sur des mots-clés correspondant précisément à sa thématique.

Exemple avec les balises de h1 à h6 :

<h1> titre 1 </h1> <h2> titre 2 </h2> <h3> titre 3 </h3> <h4> titre 4 </h4> <h5> titre 5 </h5> <h6> titre 6 </h6>

Visuellement, la balise h1 affiche du texte en "plus gros" que la balise h2. Mais attention, le HTML n’est pas fait pour mettre en forme ; ici l’objectif principal est de définir la sémantique (donner du sens) de vos pages pour les moteurs de recherche.

Avec la balise h1, nous indiquons simplement au moteur de recherche que le texte entre balises h1 est un texte très important.

Ce qu’il n’est pas

HTML n’est pas un langage de programmation. Il ne permet pas de faire des traitements et n’offre pas les instructions classiques de déclaration, d’affectation de variables, de boucles, de tests SI…ALORS…SINON… etc.

Pour « programmer » une page Web, il faudra utiliser un langage comme JavaScript, PHP ou Java par exemple.

Généralités sur l’aspect du code

  1. En HTML5, les balises ne sont pas sensibles à la casse. L’usage s’est généralisé d’écrire les balises en minuscules. On peut donc écrire les balises ainsi : <BALISE>, <Balise> ou <balise>.

  2. Toute balise ouverte <balise> doit être fermée par </balise>, mis à part les balises dites orphelines.

  3. Les balises doivent être correctement imbriquées. Lorsqu’on affecte plusieurs balises à un élément, l’ordre de fermeture est essentiel. La première balise fermante doit correspondre à la dernière balise d’ouverture non fermée.

Exemple :

Est correct : <a><b><c>contenu</c></b></a>.
Est incorrect : <a><b><c>contenu</a></c></b>.

  1. Les valeurs des attributs doivent toujours figurer entre des "guillemets".

Exemple : ici lang est l’attribut, "fr" sa valeur.

<html lang="fr">

  1. Un code doit être indenté pour être lisible. L’indentation consiste à ajouter des tabulations pour marquer qu’une balise dépend de la balise précédente. Sauter des lignes favorise la lecture du code.

Le Doctype

Le doctype, littéralement « type de document » est une instruction placée au début des pages HTML.

Cette instruction spécifie le DTD (Document Type Définition) utilisé. Le DTD définit comment les balises HTML sont formées (par exemple, si une balise peut en inclure une autre, quels attributs sont utilisables, etc.). Le DTD est indispensable aux validateurs comme celui du W3C.

Le validateur est disponible à cette URL : http://validator.w3.org/

Le doctype indique au navigateur la version HTML utilisée, ce qui permet une interprétation correcte du document.

En HTML 5

Les choses ont été grandement simplifiées concernant le doctype. Il suffit d’ajouter la ligne suivante en début de page :

<!DOCTYPE html>

Dans les versions précédentes, il fallait ajouter :

  • HTML 4.01 strict :

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

  • HTML 4.01 Transitional :

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

De quoi s’y perdre un peu.

Le Doctype ? Est-ce vraiment important ?

Certains navigateurs (Mozilla Firefox par exemple) affichent dans leur interface "le mode de rendu". Avec un doctype renseigné, on obtient "Mode de respect des standards", sinon "Mode de compatibilité (quirks)".

En l’absence d’un doctype, le navigateur applique des règles propres, ce qui peut entraîner des différences sensibles dans le rendu. Ce mode est appelé "quirks mode".

La balise <html>

La balise <html> indique au navigateur qu’il s’agit d’un document HTML.

C’est l’élément racine du document, placé juste après le doctype.

<!DOCTYPE html>
<html lang="fr">
...
</html>

La balise <html> peut prendre l’attribut lang="fr" pour spécifier la langue du document (information utile aux moteurs de recherche).

Pour plus d’informations, consultez www.w3schools.com.

L’élément <head> : l’En-tête

La balise <head>... </head> se place juste après le doctype et la balise <html>. Elle contient toutes les informations relatives au document, indépendamment du contenu affiché.

Ces informations incluent :

  • Le titre du document (balise <title>).

  • Des déclarations ou appels de feuilles de style CSS.

  • Des liens vers des fichiers JavaScript ou des scripts internes.

  • Des informations pour les moteurs de recherche (SEO).

  • Des balises meta (encodage, description, mots-clés, auteur, copyright, etc.).

En HTML5, le type des scripts, styles et liens est par défaut, il n’est donc plus nécessaire de le spécifier.

En HTML 4.0, il fallait écrire :

<script type="text/javascript"> ... </script>
<script type="text/javascript" src="fichier.js"></script>

En HTML5, on peut écrire simplement :

<script> ... </script>
<script src="fichier"></script>

De même pour le CSS :

<style> ... </style>
<link rel="stylesheet" href="fichier.css">

Notre document minimal HTML5 devient :

<!DOCTYPE html>
<html lang="fr">
<head>
...
</head>
...
</html>

Mis à part le titre, aucun élément du <head> n’apparaît dans le navigateur.

L’encodage des caractères

https://codepen.io/samuel-gibert/pen/oNGobqx

Si les caractères accentués ou spéciaux de votre page s’affichent mal, c’est généralement dû à un problème d’encodage.

<h1>Je suis content d’être en BTS SIO</h1>

Depuis fin 2011, le WC3 recommande le charset UTF-8 pour son universalité.

En 2017, 90.4 % des sites utilisent ce jeu de caractères. Pour indiquer au navigateur comment interpréter les caractères :

<meta charset="UTF-8">

En HTML 4.0, la déclaration d’encodage était :

<meta http-equiv="Content-Type" content="text/html; charset=iso-8859-1">

La balise <body>

Les balises <body>... </body> déterminent le corps du document, c’est-à-dire le contenu affiché dans le navigateur.

En HTML5, les attributs de présentation du corps (bgcolor, background, text) sont abandonnés au profit du CSS.

Un document HTML5 minimal se présente ainsi :

<!DOCTYPE html>
<html lang="fr">
  <head>
    <title>Html5</title>
    <meta charset="UTF-8">
  </head>
  <body>
    ...
  </body>
</html>

Les commentaires

Il est recommandé de commenter son code HTML pour faciliter sa compréhension lors d’une mise à jour.

Les commentaires se placent entre <!-- et --> et ne sont pas affichés dans le navigateur.

Exemple :

<h1>Je suis content d’être en BTS SIO</h1>
<!-- commentaire particulier sur plusieurs lignes --> Run

Les balises Conteneurs textes

Pour incorporer du texte dans vos pages Web, il est préférable de le placer dans des conteneurs (titres, paragraphes, listes, etc.) afin de pouvoir le mettre en forme avec le CSS.

Les Balises titre <H1> à <H6>

https://codepen.io/samuel-gibert/pen/oNGoYwG

Le HTML propose 6 niveaux de titres (h1 à h6). Le terme anglais "heading" indique leur importance décroissante.

Quelques remarques :

  • Par défaut, les balises de titre affichent le texte en gras et avec des marges (pour séparer les sections), mais leur rôle principal est de structurer la page pour le référencement.

  • Les balises de titre sont des éléments de bloc.

  • Il est important que ces balises contiennent des mots-clés pertinents pour le référencement.

  • L’attribut align (HTML 4.0) n’est plus utilisé en HTML5 ; l’alignement se fait via le CSS.

Les paragraphes : la balise <P>

Les paragraphes améliorent la lisibilité du contenu en le divisant en blocs.

Pour définir un paragraphe, on utilise la balise <p>... </p>.

Exemple :

<p>Un contenu textuel gagne en lisibilité et compréhension lorsqu’il est divisé en différents paragraphes. Pour définir un paragraphe, la balise &lt;p&gt; ... &lt;/p&gt; est utilisée. C’est un élément de type bloc avec une marge de séparation avant et après.</p> Run

À noter :

  • Les caractères réservés pour les balises (< et >) sont affichés en utilisant leurs entités HTML (&lt; et &gt;). De même pour les guillemets et le &.

  • Les retours à la ligne et tabulations dans le code HTML sont généralement ignorés (transformés en simple espace).

  • Une liste complète des entités HTML est disponible sur ce site.

  • Si le caractère est directement imprimable (é, ç, etc.), il est préférable de l’utiliser.

Exemple : Le charset est ici UTF-8

<p>caractères spéciaux :♠⓫♦↑≤</p> Run

Les listes : <OL> <UL> <LI> <DL> <DT> <DD>

Les listes sont indispensables pour structurer le contenu.

Les listes ordonnées

Une liste ordonnée se crée avec la balise <ol>... </ol> et ses items avec <li>... </li> :

<ol>
  <li>Chapitre 1</li>
  <li>Chapitre 2</li>
  <li>Chapitre 3</li>
  <li>Chapitre 4</li>
</ol> Run

  • L’attribut start permet de commencer la numérotation à un chiffre autre que 1.

  • Le nouvel attribut HTML5 reversed permet une numérotation descendante.

  • Bien que déprécié, l’attribut type définit le type de numérotation (type="1|a|À|i|I").

Exemple :

<ol start="4" type="I" reversed>
  <li><strong>Chapitre 1</strong></li>
  <li><strong>Chapitre 2</strong></li>
  <li><strong>Chapitre 3</strong></li>
  <li><strong>Chapitre 4</strong></li>
</ol> Run

  • L’attribut value permet de modifier la valeur d’un item (ex. <li value="8">Chapitre 4</li>).

Les listes non ordonnées

Les listes non ordonnées se créent avec la balise <ul>... </ul> et ses items avec <li>... </li> :

<ul>

  <li>Chapitre 2</li>
  <li>Chapitre 3</li>
  <li>Chapitre 4</li>
</ul> Run

  • L’attribut type définit la forme de la puce (disc, square, circle).

Le texte préformaté : <PRE>

La balise <pre> affiche le texte tel qu’il est saisi (espaces, tabulations, retours à la ligne sont préservés). On utilise généralement une police à largeur fixe.

La mise en forme du texte

Une fois le texte placé dans des conteneurs (par exemple, <p>), on peut appliquer divers éléments HTML de mise en forme qui ont une importance sémantique.

Mettre en gras : <B> <STRONG>

B pour Bold.

<p>Texte: <b> Du texte en gras </b></p>
<p>Texte: <strong> Du texte en gras </strong></p> Run

Visuellement, il n’y a pas de différence, mais la balise <strong> apporte une information sémantique utile aux lecteurs d’écran.

La mise en italique : <i> <em>

Exemple : il est possible de combiner du texte en italique et en gras.

<p><i>Du texte en italique seul</i></p>
<p><b><i>Du texte en italique et en gras</i></b></p> Run

La balise <em> (emphase) met également le texte en italique de façon sémantique.

La mise en indice et en exposant : <SUP> <SUB>

Pour les formules mathématiques ou chimiques, <sup> affiche le contenu en exposant et <sub> en indice.

<p>(a + b)<sup>2</sup> = a<sup>2</sup> + 2ab + b<sup>2</sup>
<br> H<sub>2</sub>O</p> Run

Le souligné <u> et <INS>, le barré <S> et <del>

La balise <u> permet de souligner sans ajouter de signification sémantique. <ins> indique un contenu inséré, tandis que <s> ou <del> indiquent un contenu barré ou supprimé.

<p>de ce fait, le W3C suggère d’en faire un usage <s>systématique</s><u>judicieux</u>.</p>
<p>de ce fait, le W3C suggère d’en faire un usage <del>systématique</del> <ins>judicieux</ins>.</p> Run

Réduire la taille d’un texte : <small>

La balise <small> permet d’afficher du texte en plus petit, indiquant une moindre importance.

<p>voici le contrat: <small> ce n’est pas parce que c’est écrit petit que je ne dois pas être lu... </small></p> Run

Sens de lecture : <BDO>

Pour les écritures de droite à gauche, l’attribut dir peut être défini (ltr ou rtl). Placé dans une balise <bdo>, il inverse l’ordre d’apparition des lettres.

<p dir="rtl">L’attribut dir permet de définir le sens de lecture</p>
<p><bdo dir="rtl">L’attribut dir permet de définir le sens de lecture</bdo></p>
<p dir="rtl"><bdo dir="rtl">L’attribut dir permet de définir le sens de lecture</bdo></p> Run
