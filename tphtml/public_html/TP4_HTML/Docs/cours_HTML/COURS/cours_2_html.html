HTML 5 – Partie 2

Les liens Hypertextes : balise <a>

L’essence même du Web est de pouvoir naviguer sur différentes pages en suivant des liens qui peuvent être :

  • Un endroit de la page en cours.
  • Une autre page du site.
  • Un endroit dans une autre page du site.
  • Une page d’un autre site situé sur le Web.
  • Une adresse électronique.
  • Un fichier à télécharger.

Les liens apparaissent par défaut soulignés et de couleur bleue pour un lien non visité, de couleur pourpre pour un lien visité et rouge pour un lien actif. Le soulignement est une convention majeure du Web pour indiquer les liens hypertextes. Il est toutefois possible de modifier leur style en CSS.

La balise utilisée est la balise <a>, qui signifie ancre.

Comprendre les chemins absolus et relatifs

Le protocole principal utilisé pour le Web est HTTP (Hyper Text Transfer Protocol), basé sur le modèle client-serveur dont le fonctionnement se résume ici :

  1. Le client demande une page via son navigateur (Chrome, Firefox, etc.). Par exemple, l'utilisateur demande la page http://www.example.com/test.html.

    Cette URL se compose de 2 parties :

    • Le serveur : http://www.example.com

    • La page demandée : /test.html

    La requête s'exprime ainsi : www.example.com GET /test.html.

    Le / correspond à la racine du serveur.

  1. Le serveur reçoit la requête, vérifie la présence de la page demandée et la renvoie au client, qui interprète le code HTML et affiche la page.

Une erreur dans le format d’écriture des liens peut entraîner des pages introuvables, des images manquantes ou des styles non appliqués. Par exemple, référencer une page par rapport à un chemin physique (ex. : C:\Inetpub\wwwroot\test.html) est incorrect.

Deux types d’écritures cohabitent :

  • Les chemins absolus : chemin d’accès complet (commence par http:// ou par la racine /).

  • Les chemins relatifs : chemin relatif au répertoire courant de la page. ".." indique le répertoire parent et "." le répertoire courant.

Les différents types de liens

Lien vers une autre page du même site

Les liens vers une autre page du site se font généralement avec des liens relatifs. On utilise l’attribut href pour définir la cible.

Exemple :

<!-- Lien vers page2.htm dans le même répertoire -->
<a href="page2.htm"><strong>Lien vers page 2</strong></a>
<a href="./page2.htm"><strong>Lien vers page 2</strong></a>

<!-- Lien vers page1.htm dans dossier1 du même répertoire -->
<a href="dossier1/page1.htm"><strong>Texte du lien</strong></a>
<a href="./dossier1/page1.htm"><strong>Texte du lien</strong></a>

<!-- Lien vers page2.htm dans dossier2 du répertoire parent -->
<a href=".. /dossier2/page2.htm"><strong>Texte du lien</strong></a>

<!-- Lien vers index.htm dans le répertoire parent -->
<a href=".. /index.htm"><strong>Texte du lien</strong></a>

La balise <a> peut contenir d’autres éléments comme des images, des titres, etc.

Exemple :

<a href="page2.htm">
  <div>
    <p>Les liens vers une autre page se font avec des liens relatifs et l’attribut href détermine la cible.</p>
  </div>
</a>

Le lien à l’intérieur d’une même page

Pour accéder à un endroit précis d’une page (ancre), on définit un attribut id :

<a id="nomAncre"></a>

Pour lier à cette ancre, on utilise le caractère # suivi du nom :

<a href="#nomAncre">Texte du lien</a>

Exemple :

<a href="#ch2"> <strong>Accès aux chapitres 2</strong></a>
<a id="ch1"></a> <!-- ⚓ -->
<h1><strong>Chapitre 1</strong></h1>
<p><strong>Le texte du chapitre 1</strong>
...
</p>
<a id="ch2"></a> <!-- ⚓ -->
<a href="#ch1"> <strong>Retour au chapitre 1</strong></a>
<h1><strong>Chapitre 2</strong></h1>

Il est aussi possible de référencer une ancre d’une autre page :

<a href="dossier1/page1.htm#ancre">Texte du lien</a>

Une ancre par défaut #top (ou simplement #) permet de retourner en haut de la page :

<a href="#top">Retourner en haut de la page</a>
<a href="#">Retourner en haut de la page également</a>

Le lien vers un autre site (externe)

Pour pointer vers une page d’un autre site, on utilise une URL absolue (n’oubliez pas le http://) :

<a href="http://www.w3.org">Lien vers W3C</a>

Le lien vers une adresse email

Pour créer un lien vers une adresse email :

<a href="mailto:samuel.gibert@gmail.com">M’envoyer un mail</a>

Le clic ouvre l’application de messagerie par défaut.

La plupart des navigateurs permettent de modifier l’application mail par défaut.

Il est aussi possible de prédéfinir l’objet ou ajouter une copie :

  • Pour prédéfinir un objet :

<a href="mailto:samuel.gibert@gmail.com?subject=Html5"> m’envoyer un mail</a>

  • Pour définir une copie :

<a href="mailto:samuel.gibert@gmail.com?cc=autres@gmail.com"> m’envoyer un mail</a>

  • Pour combiner :

<a href="mailto:samuel.gibert@gmail.com?subject=Html5&cc=autres@gmail.com">m’envoyer un mail</a>

Lien vers un numéro de téléphone

Pour un smartphone, il est possible de créer un lien vers un numéro de téléphone :

<a href="tel:+600000000">+6 15 70 15 64</a>

Le lien pour télécharger un fichier

Pour télécharger un fichier, spécifiez le nom dans href. Par défaut, le navigateur essaiera de l’ouvrir. Avec l’attribut download, vous forcez le téléchargement.

<a href="./pdf/Cours_HTML_5.pdf" target="_blank"> Ouvrir le cours </a>
<a href="./pdf/Cours_HTML_5.pdf" target="_blank" download> Téléchargez le cours </a>

Pour un fichier PDF, le navigateur l’ouvrira par défaut. Pour d’autres fichiers, s’il n’existe pas d’application par défaut, le fichier sera téléchargé. Dans ce cas, l’attribut download est facultatif.

<a href="./archive/Cours_HTML_5.zip"> téléchargez le cours </a><br>

Attributs utiles pour la balise <a>

Outre href et target, d’autres attributs sont utiles :

  • title="valeur" : affiche une info-bulle.

  • download : force le téléchargement.

  • hreflang="valeur" : indique la langue de la ressource liée.

  • type : indique le type MIME de la ressource (indicatif).

Conseils de mise en œuvre

Ce chapitre résume un article avec un bon et un mauvais exemple pour chaque conseil :

https://developer.mozilla.org/fr/docs/Apprendre/HTML/Comment/Cr%C3%A9er_un_hyperlien

  • Le texte d'un lien doit être compréhensible seul :

<a href="https://firefox.com/"> Télécharger Firefox </a>
<del><a href="https://firefox.com/"> Cliquez ici</a> pour télécharger Firefox</del>

  • N'ajoutez pas "lien vers" dans le texte du lien :

<a href="https://developer.mozilla.org/Firefox_OS"> En savoir plus sur Firefox OS</a>
<del>Voici un <a href="https://developer.mozilla.org/Firefox_OS"> lien sur Firefox OS </a></del>

  • Indiquez la cible du lien plutôt que de copier-coller une URL :

<a href="https://careers.mozilla.org/university/"> Candidatez pour un stage chez Mozilla</a>
<del><a href="https://careers.mozilla.org/university/"> https://careers.mozilla.org/university </a></del>

  • Le texte d'un lien doit être concis :

Voici une page où vous pouvez <a href="https://www.mozilla.org/firefox/all/"> télécharger Firefox </a>
<del>Voici une page où <a href="https://www.mozilla.org/firefox/all/"> vous pouvez télécharger Firefox en 90 langues différentes, pour quatre systèmes d'exploitation différents</a></del>

Le style sur les liens

Ce cours présente les propriétés CSS permettant de mettre en forme les liens HTML. Les mêmes propriétés s'appliquent aux liens comme au texte.

Pour modifier le style, il faut comprendre les pseudoclasses en CSS.

Sélecteur avec pseudoclasses

Les pseudoclasses permettent de modifier le style d’un élément selon son état (par exemple, :hover pour le survol).

Sélecteur : pseudoclasse { déclaration(s) de style }
Exemple :
div:hover {
   background-color: #F89B4D;
}

Liste des pseudoclasses standard : https://developer.mozilla.org/fr/docs/Web/CSS/Pseudo-classes

HTML CSS externe

<a href="http://www.google.fr" target="_blank"> vers google </a>

a:link {
   color: red;
   text-transform: uppercase;
}
a:visited {
   text-decoration: underline red;
   color: blue;
}
a:hover {
   font-weight: bold;
   font-style: italic;
   font-size: 1.5em;
}
a:active {
   text-decoration: underline overline green;
}

Pseudos classes de paragraphe

  • :first-letter définit l’apparence de la première lettre d’un paragraphe.

  • :first-line définit l’apparence de la première ligne d’un paragraphe.

Exemple :

HTML CSS externe

<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit,
sed do eiusmod tempor incididunt ut … anim id est laborum.</p>

p:first-letter {
   text-transform: uppercase;
   font-size: 1.5em;
   color: blue;
}
p::first-line {
   font-weight: bold;
}

Les pseudo-classes d’insertion

  • :before insère du contenu avant un élément.

  • :after insère du contenu après un élément.

Exemple :

HTML CSS externe

<p class="tel"> 06012345 67 </p>
<p class="mail"> samuel.gibert@ac-grenoble.fr </p>

.tel:before {
   content: ' ✆- ';
}
.mail:after {
   content: ' -📧 ';
}
a:hover:before {
   content: '⇨';
}
